# MCP SuperAssistant - 独立运行配置

此文档说明了如何使 MCP SuperAssistant 的本地代理服务独立于 VSCode 运行。

## 功能

- ✅ **独立运行**：无需打开 VSCode 或其他开发工具。
- ✅ **两种启动模式**：提供带窗口的调试模式和无窗口的后台静默模式。
- ✅ **开机自启**：可配合 `setup_startup.ps1` 脚本实现开机自动后台运行。

## 快速开始

### 1. 环境准备

确保你的系统已安装 **Node.js** (>= 22.12.0)。
- 从官网下载并安装: https://nodejs.org/
- 确保 Node.js 已添加到系统的 PATH 环境变量中。

### 2. 手动启动

你有两种方式来启动本地代理服务器：

#### A. 后台静默启动 (日常使用推荐)

双击 `start_silent.vbs` 文件。

这将在后台启动服务器，不会弹出任何命令行窗口。浏览器扩展将能正常连接和工作。

#### B. 带窗口启动 (用于调试)

双击 `start_standalone.bat` 文件。

这将打开一个命令行窗口并启动服务器。你可以实时看到所有的日志输出，方便排查问题。关闭此窗口即可停止服务。

### 3. (可选) 设置开机自动启动

如果你希望每次开机时服务都能自动在后台运行，请执行以下操作：

1.  右键点击 `setup_startup.ps1` 文件。
2.  选择 “使用 PowerShell 运行”。
3.  如果系统提示权限问题，请尝试以管理员身份运行 PowerShell，然后执行 `.\setup_startup.ps1`。

此脚本会在 Windows 的“启动”文件夹中创建一个指向 `start_silent.vbs` 的快捷方式，从而实现开机自启。

## 如何停止后台服务

由于后台静默启动没有窗口，你需要通过 **任务管理器** 来手动停止它：

1.  按 `Ctrl + Shift + Esc` 打开任务管理器。
2.  在 "进程" 标签页中，找到名为 **`Node.js: Server-side JavaScript`** 的进程 (可能会有多个，请注意辨别)。
3.  选中它，然后点击右下角的“结束任务”。

## 脚本文件说明

-   `start_standalone.bat`:
    带命令行窗口的启动脚本。它会检查 Node.js 环境，然后执行 `start.js` 来启动服务。主要用于调试。

-   `start_silent.vbs`:
    无窗口的静默启动脚本。它会调用 `start_standalone.bat`，但会隐藏其命令行窗口。推荐日常使用此方式。

-   `setup_startup.ps1`:
    一个 PowerShell 脚本，用于方便地创建或移除开机自启的快捷方式。

-   `start.js` & `start_proxy.js`:
    这两个是 Node.js 脚本，是启动代理服务器的核心逻辑。它们负责执行 `npx` 命令来拉取并运行最新的 `@srbhptl39/mcp-superassistant-proxy` 包。

## 故障排除

### 问题：双击脚本后，服务没有运行，浏览器扩展依然报错。

**解决方案：**
1.  尝试使用 `start_standalone.bat` 启动，查看命令行窗口中是否有红色的错误信息。
2.  最常见的问题是 Node.js 没有安装或没有被正确添加到系统环境变量 PATH 中。请根据脚本的提示进行检查。
3.  确保您的网络连接正常，因为首次运行时脚本需要通过 `npx` 从网络下载代理服务器的包。
